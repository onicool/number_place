# iPad向け Number Place ゲーム 設計書

## 1. 目的

子供でも迷わず遊べる、iPad向けの Number Place（ナンプレ）ゲームを自作する。
最優先は「操作感のよさ」であり、以下を重視する。

- タップしやすい
- 何をすればよいか分かりやすい
- 間違えてもストレスが少ない
- すぐに遊び始められる

## 2. 想定ユーザー

- 主対象: 小学生前後の子供
- 副対象: 子供と一緒に遊ぶ保護者

ユーザー特性:

- 細かいUIよりも大きく分かりやすい操作を好む
- ルール説明は短く、視覚的な誘導が必要
- 失敗時に強い否定表現があると離脱しやすい

## 3. コンセプト

「やさしく触れて、すぐ分かるナンプレ」

難しい機能を増やすより、以下の体験を優先する。

- 盤面を押した時に気持ちよく反応する
- 入れられる数字がすぐ分かる
- 間違いに気づきやすい
- クリアまでの達成感がある

## 4. 対応プラットフォーム

### 4.1 第一候補

iPad Safari で動く Web アプリ

理由:

- App Store 公開なしでも試作しやすい
- 配布と更新が簡単
- 将来的に PWA 化しやすい

### 4.2 将来対応

- PWA 化してホーム画面追加に対応
- 必要ならネイティブラッパー化

## 5. ゲーム方針

### 5.1 初期リリースの範囲

初期版は機能を絞る。

- 9x9 の標準ナンプレ
- 難易度は「かんたん」中心
- 1問を気持ちよく解けることを重視
- 強い競争要素は入れない

### 5.2 子供向けの簡単化

- 同じ数字の重複を入力時にすぐ色で知らせる
- 選択中のマスと同じ行・列・ブロックをハイライトする
- 同じ数字もやさしくハイライトする
- 行き詰まりにくいようにヒント機能を用意する
- 初期版では高難度問題は出さない

### 5.3 拡張候補

- 4x4 のれんしゅうモード
- 6x6 のステップアップモード
- 日替わり問題
- ごほうび演出

## 6. UX設計方針

### 6.1 最重要UX

1回の操作を短く、迷いなく終えられること。

理想フロー:

1. マスをタップする
2. 画面下の大きな数字ボタンから 1 つ選ぶ
3. すぐ盤面に反映される
4. 間違いがあればその場でやさしく伝える

### 6.2 操作原則

- 主要操作はすべて片手タップで完結させる
- 長押し前提の操作は作らない
- 小さなアイコンよりラベル付きボタンを優先する
- 誤タップしても戻しやすくする

### 6.3 タッチ設計

- 盤面セルは iPad 縦持ち時でも十分な大きさを確保する
- 数字ボタンは最低 56px 以上を目安にする
- ボタン間隔を確保して誤タップを減らす
- タップ時に縮小や色変化で即時反応を返す

## 7. 画面構成

### 7.1 ホーム画面

要素:

- タイトル
- 「はじめる」ボタン
- 難易度選択
- つづきから
- あそびかた

方針:

- 1画面で完結する
- 最初に押すべきボタンを目立たせる

### 7.2 ゲーム画面

要素:

- 上部: 戻る、現在の難易度、タイマー、ヒント
- 中央: 盤面
- 下部: 数字入力パッド 1〜9
- 補助操作: けす、ひとつもどす

方針:

- 盤面を主役にする
- 数字入力パッドは常に固定表示する
- モード切替を減らして覚えることを少なくする

### 7.3 クリア画面

要素:

- 「できた！」などの達成メッセージ
- 星やスタンプ風の演出
- 次の問題へ
- もう一回あそぶ

方針:

- 褒める演出はするが長すぎない
- すぐ次へ進める

## 8. 詳細操作仕様

### 8.1 マス選択

- タップしたマスを選択状態にする
- 選択マスは枠線と背景色で明確に示す
- 同じ行・列・3x3 ブロックも薄くハイライトする
- 同じ数字が盤面上にあればそれも淡く光らせる

### 8.2 数字入力

- 下部の数字ボタンを押すと選択マスに入力される
- 最初から入っている固定数字は編集不可
- すでに数字があるマスは上書き可能にする

### 8.3 間違い表示

- 同じ行・列・ブロックに重複がある場合、入力直後に色で示す
- 強いエラー音や大きな警告ダイアログは使わない
- 表示文言は「ちがうかも」など柔らかいものにする

### 8.4 取り消し

- 「ひとつもどす」で直前操作を戻せる
- 子供向けのため、戻す回数は実質無制限でよい

### 8.5 削除

- 「けす」で選択マスを空にできる
- 固定数字は削除不可

### 8.6 ヒント

初期版のヒントは複雑にしすぎない。

- 選択マスに入る正しい数字を1つ教える
- または、埋められるマスを1つハイライトする

ヒント利用時の方針:

- 回数制限は付けない、または緩くする
- 子供が詰まって止まることを防ぐ

## 9. 難易度設計

### 9.1 初期提供難易度

- かんたん
- ふつう

ただし、初期の主軸は「かんたん」に置く。

### 9.2 難易度定義

かんたん:

- 空きマス少なめ
- 早い段階で埋められる場所が見つかる
- ヒントなしでも進めやすい

ふつう:

- 少し考える場面が増える
- ただし子供が投げ出す難しさにはしない

### 9.3 出題品質

- 必ず一意解を持つ問題のみ採用
- 初手から極端に詰まりやすい問題は除外
- クリアまでの想定時間は 5〜15 分程度を目安にする

## 10. ビジュアル方針

### 10.1 デザインの方向性

- 明るくやさしい配色
- 高コントラストで見やすい数字
- 情報量を増やしすぎない

### 10.2 色の考え方

- 通常背景: 明るいニュートラル色
- 選択中セル: はっきり分かるアクセント色
- 関連領域: 薄い補助色
- エラー: 強すぎない暖色
- 完了: 楽しい達成色

### 10.3 文字と数字

- 数字は丸みがあり判別しやすいフォントを使用する
- 細すぎるウェイトは避ける
- 固定数字とユーザー入力数字は色で区別する

## 11. サウンド・演出

### 11.1 サウンド

- タップ音は短く軽いもの
- エラー音は不快感のない控えめなもの
- クリア時に短い成功音を鳴らす

### 11.2 アニメーション

- タップ時の反応は 100〜150ms 程度
- クリア演出は 1〜2 秒で終わる
- 盤面操作の邪魔になる長い演出は避ける

## 12. 機能要件

### 12.1 必須機能

- 問題の表示
- マス選択
- 数字入力
- 入力削除
- 取り消し
- 重複チェック表示
- クリア判定
- ヒント
- 一時保存

### 12.2 あるとよい機能

- チュートリアル
- 効果音 ON/OFF
- タイマー表示 ON/OFF
- 実績やスタンプ

### 12.3 初期版では後回しにする機能

- オンラインランキング
- 難問モード
- アカウント連携
- 広告最適化

## 13. 非機能要件

### 13.1 パフォーマンス

- 画面操作に対する反応は体感で即時にする
- 数字入力後の反映は遅延なく行う
- iPad の一般的なブラウザで快適に動作する

### 13.2 使いやすさ

- 主要ボタンは片手でも押しやすい位置に置く
- 画面回転は初期版では縦持ち優先でよい
- 子供が読める短い文言にする

### 13.3 保存

- 途中経過をローカル保存する
- アプリを閉じても続きから再開できる

## 14. 推奨技術構成

### 14.1 フロントエンド

- HTML / CSS / JavaScript
- または React + TypeScript

推奨:

- React + TypeScript

理由:

- 盤面状態の管理がしやすい
- 将来の機能追加に対応しやすい
- コンポーネント単位でUIを整理しやすい

### 14.2 レイアウト

- CSS Grid で盤面を構築する
- 下部入力パッドは固定配置にする
- iPad の safe area を考慮する

### 14.3 データ管理

- 問題データは JSON で持つ
- セーブデータは localStorage で保存する

## 18. 受け入れ基準

- iPad で縦持ちしたときに盤面と数字ボタンが無理なく表示される
- 子供が説明なしでも「マスを選んで数字を押す」操作を理解しやすい
- 誤入力してもすぐ戻せる
- 1問を短時間で気持ちよく終えられる
- 少なくとも「かんたん」難易度で詰まりにくい

## 19. リスクと対策

### リスク1: 子供には 9x9 が難しすぎる

対策:

- 将来的に 4x4 練習モードを追加する
- 初期問題はかなり易しめに調整する
- ヒントを強めにする

### リスク2: タッチ誤操作が多い

対策:

- タップ領域を大きくする
- 下部入力パッドを固定する
- 取り消しを目立つ位置に置く

### リスク3: 見た目は良いが遊びにくい

対策:

- 盤面中心のレイアウトを維持する
- 装飾を増やしすぎない
- 実機で操作テストする

## 20. まとめ

このアプリでは、ナンプレの複雑さよりも「触って分かる」「気持ちよく進める」体験を優先する。
初期版は、iPad向けWebアプリとして以下を満たすことを目標とする。

- 大きく押しやすいUI
- やさしいエラー表示
- 簡単な問題中心
- 子供が止まりにくいヒント設計
- 途中再開できる保存機能

まずは最小構成で遊べる版を作り、その後に演出や練習モードを追加して完成度を上げていく。
